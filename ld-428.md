## 项目主要需求

### 项目概述

【全栈Web应用】艺术作品集与画廊管理平台。面向独立艺术家和小型画廊，支持作品管理、线上展览策划、观众互动（点赞/评论/收藏）和艺术家个人主页。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **作品（Artwork）** — 字段：作品标题、作品描述、创作年份、媒介（OilPainting/Watercolor/Acrylic/Ink/Sculpture/Photography/Digital/Mixed/Installation/Other）、材质、尺寸（长×宽×高cm）、图片URL列表、视频URL（可选）、标签列表、艺术家ID、所属展览ID列表、作品状态（Draft/Published/Sold/Archived）、价格（可选）、浏览数、点赞数、收藏数
   - 业务动作：上传作品、编辑信息、发布/下架
   - **贯穿全栈**

2. **展览（Exhibition）** — 字段：展览标题、展览描述、策展人ID、开始日期、结束日期、展览类型（Solo/Group/Thematic/Permanent）、展览封面URL、作品ID列表、展览状态（Planning/Active/Ended/Archived）、参观人数
   - 业务动作：创建展览、添加/移除作品、发布展览
   - **贯穿全栈**

3. **艺术家（Artist）** — 字段：用户ID、艺术家名、个人简介、头像URL、擅长媒介列表、代表作品URL列表、社交媒体链接、关注者数、粉丝数、状态（Active/Inactive）
   - 关联：1个艺术家 → 多个作品
   - **贯穿全栈**

4. **互动记录（Interaction）** — 字段：用户ID、目标类型（Artwork/Exhibition）、目标ID、互动类型（Like/Comment/Bookmark/Share）、评论内容（评论时有）、创建时间
   - 业务动作：点赞、评论、收藏、取消
   - **贯穿全栈**

5. **审核日志（ReviewLog）** — 字段：目标类型（Artwork/Exhibition）、目标ID、审核人ID、审核结果（Approved/Rejected/Flagged）、审核意见、审核时间
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **画廊首页** `/gallery` — 消费：Artwork + Exhibition
   - 精选展览轮播、最新作品瀑布流、热门作品排行
   - 使用 `<ArtworkCard>`, `<ExhibitionBanner>`, `<EmptyState>`

2. **作品详情** `/artwork/:id` — 消费：Artwork + Interaction + Artist
   - 大图浏览（支持缩放）、作品信息面板、点赞/收藏按钮、评论列表
   - 使用 `<ArtworkCard>`, `<CommentSection>`, `<InteractionBar>`

3. **展览详情** `/exhibition/:id` — 消费：Exhibition + Artwork + Artist
   - 展览封面大图、展览描述、作品网格列表、策展人信息
   - 使用 `<ArtworkCard>`, `<ExhibitionBanner>`

4. **艺术家主页** `/artist/:id` — 消费：Artist + Artwork + Exhibition
   - 艺术家头像和简介、代表作品轮播、作品集列表、参与展览
   - 使用 `<ArtworkCard>`, `<StatusBadge>`, `<UserAvatar>`

5. **创作工作台（艺术家后台）** `/studio` — 消费：Artwork + Exhibition
   - 我的作品管理（草稿/已发布/已售Tab）、上传新作品、创建展览、数据统计（浏览/点赞/收藏趋势）
   - 使用 `<StatusBadge>`, `<StatCard>`, `<EmptyState>`

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / Curator / Artist / Viewer
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮显隐（Viewer只能浏览和互动）+ 类型定义

2. **内容审核**
   - 艺术家提交的作品和展览需要审核后才公开
   - 触达文件层：DB review_log表 + 审核中间件 + 审核service + 前端审核页面 + 类型定义

3. **操作日志**
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 类型定义

### 共享枚举/常量（≥ 2 组）

1. **作品状态 ArtworkStatus**：Draft / Published / Sold / Archived
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`

2. **媒介类型 Medium**：OilPainting / Watercolor / Acrylic / Ink / Sculpture / Photography / Digital / Mixed / Installation / Other

3. **展览状态 ExhibitionStatus**：Planning / Active / Ended / Archived

4. **互动类型 InteractionType**：Like / Comment / Bookmark / Share

### 共享前端组件清单（≥ 3 个）

1. **`<ArtworkCard>`** — 画廊首页、展览详情、艺术家主页、创作工作台
2. **`<ExhibitionBanner>`** — 画廊首页、展览详情
3. **`<CommentSection>`** — 作品详情
4. **`<InteractionBar>`** — 作品详情（点赞/收藏/分享按钮组）
5. **`<UserAvatar>`** — 作品详情、艺术家主页

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useInteraction`** — 封装点赞/收藏/评论的乐观更新逻辑
2. **`usePagination`** — 通用分页（作品列表用无限滚动）
3. **`formatArtworkSize`** — 尺寸格式化工具

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色校验
3. **`auditLogMiddleware`** — 操作日志
4. **`contentReviewMiddleware`** — 内容审核拦截（未审核内容不公开返回）
5. **`errorHandlerMiddleware`** — 统一异常
6. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`art-gallery`
- **前端端口**：18808 / 80
- **后端端口**：19308 / 3000
- **前端**：React 18 + TypeScript + Vite
- **UI 库**：Tailwind CSS + Headless UI
- **图表**：ECharts（趋势图）
- **图片懒加载**：react-lazyload
- **后端**：NestJS + TypeScript
- **数据库**：MongoDB 7（文档型更适合艺术品管理）
- **ODM**：Mongoose
- **认证**：JWT
- **状态管理**：Zustand
- **构建工具**：Vite

### 部署要求

Docker Compose 一键部署。
- 顶层 `name: art-gallery`
- 不写 `version:` 字段
- MongoDB 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

### Git 要求

`git init` + `.gitignore`。
